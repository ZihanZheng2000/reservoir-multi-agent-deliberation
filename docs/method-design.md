# Method Design

## Agent Groups

The deliberation system uses three broad groups:

1. **Expert agents**: hydrology, dam safety, ecology, climate risk, economics/finance, social/resettlement, legal/policy.
2. **Stakeholder agents**: developer, government, local government, upstream affected residents, downstream users, environmental NGO, financier.
3. **Governance agents**: moderator, evidence auditor, decision authority.

## Role Limitation

Agents are intentionally role-limited. A hydrologist should not become a legal expert; an economist should not decide cultural acceptability; an affected-resident agent should represent affected interests rather than become a generic analyst. This limitation is part of the method because it creates meaningful disagreement.

## Run Types

### Full Run

A full run includes separate expert assessments, stakeholder assessments, a moderator conflict map, targeted second-round deliberation, evidence audit, final decision recommendation, baseline outputs, and evaluation.

### Compact Run

A compact run preserves the same logical components in compressed form. It is cheaper and faster, but it provides weaker evidence of deliberative process quality.

## Baselines

The benchmark compares multi-agent deliberation against:

- a single-agent decision analyst baseline;
- a weighted MCDA baseline.

The baselines are not expected to score on Process Quality because they are not deliberative systems.
